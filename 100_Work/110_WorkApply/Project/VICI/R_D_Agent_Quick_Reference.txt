FIVE-UNIT ARCHITECTURE OVERVIEW:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

❶ SPECIFICATION UNIT
   Input: Scenario description + task context
   Output: Formal specification S = (B, D, F, M)
   Key: Standardizes interface across all units

❷ SYNTHESIS UNIT  
   Input: Historical hypotheses H_t + feedback F_t
   Process: h_(t+1) = G(H^(a)_t, F^(a)_t)
   Output: New hypothesis + idea candidates
   Key: Knowledge Forest + SOTA tracking

❸ IMPLEMENTATION UNIT
   Input: Hypothesis h from Synthesis
   Agent: Co-STEER (code generation for quant finance)
   Output: Executable Python code for factor/model
   Key: Domain-specific + Qlib-native

❹ VALIDATION UNIT
   Input: Code from Co-STEER
   Process: Backtest on historical data
   Output: Performance metrics + updated SOTA pool
   Key: Deduplication via similarity metrics

❺ ANALYSIS UNIT
   Input: Experimental results
   Process: Evaluate + contextual bandit scheduling
   Output: Feedback signal + next action (factor vs model)
   Key: Contextual Thompson sampling (CLTS)

CLOSED LOOP: ❶→❷→❸→❹→❺→(feedback)→❷

================================================================================
CORE INNOVATIONS BREAKDOWN:
================================================================================

1. CO-STEER CODE GENERATION AGENT
   • Specialized LLM for quantitative finance code
   • Understands Qlib API, factor conventions, pandas optimization
   • Generates task-specific implementations
   • Includes error handling & validation

2. KNOWLEDGE FOREST
   • Persistent memory organized by optimization direction
   • Maintains: hypotheses (H_t), feedbacks (F_t), SOTA solutions
   • Enables cumulative learning across experiments
   • Action-conditioned filtering: F^(a)_t, H^(a)_t

3. CONTEXTUAL BANDIT SCHEDULER
   • Algorithm: Contextual Linear Thompson Sampling (CLTS)
   • Selects: Optimize factors OR optimize models
   • State-dependent: Observes current IC, Sharpe, MDD
   • Learns: Which direction yields better returns

4. JOINT FACTOR-MODEL CO-OPTIMIZATION
   • NOT separate optimization pipelines
   • Coordinated through feedback loop
   • Bandit learns which branch to prioritize
   • Synergistic: factors provide signal, models manage risk

================================================================================
KEY METRICS DEFINITIONS
================================================================================

IC (Information Coefficient)
  = Correlation(predicted_returns, actual_returns)
  Range: [-1, 1], higher is better
  ~0.02 typical for good factors

ICIR (IC Information Ratio)
  = IC / std(IC)
  Measures signal consistency
  Higher = more stable signal

ARR (Annualized Return Ratio)
  = Annualized portfolio return
  Typically reported as decimal (0.1982 = 19.82%)

Sharpe Ratio
  = (Portfolio_Return - Risk_Free_Rate) / Volatility
  Higher = better risk-adjusted return

MDD (Maximum Drawdown)
  = Largest peak-to-trough decline
  Reported as negative (-0.0656 = -6.56%)
  Lower (less negative) = better

CR (Calmar Ratio)
  = ARR / |MDD|
  Return per unit of drawdown
  Higher = better risk-return tradeoff
